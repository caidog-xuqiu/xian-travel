# 下一阶段路线图

## A. 近期（最值得继续做）
- 持续增强解析稳定性（复杂口语、冲突语义、噪声文本）
- 稳定 selector 命中率（减少真实 API 抖动导致的回退）
- 固定回归样本集与指标看板（parsed_by / selected_by / fallback 次数）
- 打磨夜游、轻松、含餐等高频场景的结果一致性

## B. 中期（有价值但不急）
- 引入节假日与人流量策略（先规则版，再接数据源）
- 增强 skills 显式化（把策略与动作能力模块化）
- 逐步接入 RAG（知识问答与背景解释，不替代硬约束）
- 优化持久化层（更完善的 thread 历史与回放能力）

## C. 后期（平台化/高级化）
- MCP 接入（外部能力统一编排）
- Multi-agent 架构探索（任务分工与协作）
- 更复杂路线优化（更强全局搜索/重排能力）
- 更完整工程化（监控、告警、发布与灰度体系）

## 说明
- 当前建议坚持“硬约束稳定优先，LLM 决策增强其次”的迭代顺序。
- 不建议在主链未完全稳定前引入过多平台化复杂度。
